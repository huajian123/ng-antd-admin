对比

核心区别：computed 只读，linkedSignal 可写但 source 变化时自动同步。

{{ baseValue() }} {{ computedDouble() }} {{ linkedDouble() }} 
computedDouble = computed(() => baseValue() * 2);
// ❌ computedDouble.set(x) — 编译错误，只读

linkedDouble = linkedSignal(() => baseValue() * 2);
// ✅ linkedDouble.set(999) — 可写
// ✅ baseValue 变化时自动重新同步

本项目 ant-table.component.ts 中的真实用法：父组件传入 tableConfig，子组件用 linkedSignal 做本地副本，可本地修改而不影响父组件。

readonly tableConfig = input.required<AntTableConfig>();
readonly _tableConfig = linkedSignal(() => this.tableConfig());
