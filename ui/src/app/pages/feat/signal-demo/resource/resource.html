resource()

request signal 变化时自动重新加载，无需手动订阅。

用户 ID：
@if (userResource.isLoading()) { } @else if (userResource.error()) { 错误：{{ userResource.error() }} } @else if (userResource.value()) { {{ userResource.value()!.name }} {{ userResource.value()!.email }} {{ userResource.value()!.role }} } 
readonly userId = signal(1);
readonly userResource = resource({{ '{' }}
          params: () => this.userId(),
  loader: ({{ '{' }} params: id {{ '}' }}) => fetchUser(id)
          {{ '}' }});
// userId 变化 → 自动重新加载
状态管理

resource 有三种状态：loading / loaded / error，通过 isLoading()、value()、error() 读取。

当前状态： @if (userResource.isLoading()) { loading } @else if (userResource.error()) { error } @else { loaded } 
// 读取三种状态
userResource.isLoading() // boolean
userResource.value()     // T | undefined
userResource.error()     // unknown

// 状态流转
// signal 变化 → loading → loaded/error
reload()

调用 reload() 强制重新加载，即使 request signal 未变化。

上次刷新： {{ lastLoadTime() }}
加载状态： @if (userResource.isLoading()) { 加载中... } @else { 已完成 } 
// 手动触发重新加载
userResource.reload();

// 适用场景：
// - 用户点击刷新按钮
// - 轮询数据
// - 操作完成后刷新列表
rxResource()

rxResource() 与 resource() 用法相同，区别是 loader 返回 Observable 而非 Promise。

用户 ID：
@if (rxUserResource.isLoading()) { } @else if (rxUserResource.value()) { {{ rxUserResource.value()!.name }} {{ rxUserResource.value()!.email }} } 
// resource() — Promise loader
readonly r1 = resource({{ '{' }}
          params: () => this.id(),
  loader: ({{ '{' }} params: id {{ '}' }}) => fetchUser(id)
          {{ '}' }});

// rxResource() — Observable loader
readonly r2 = rxResource({{ '{' }}
          params: () => this.id(),
  loader: ({{ '{' }} params: id {{ '}' }}) =>
    http.get<User>(`/api/user/${{ '{' }}id{{ '}' }}`)
          {{ '}' }});
