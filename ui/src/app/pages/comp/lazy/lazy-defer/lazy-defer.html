@defer 延迟加载

@defer 将组件的 JS 代码拆分为独立 chunk，仅在触发条件满足时才下载并渲染。 支持 on idle、on viewport、on interaction、 on hover、on timer、when 六种触发方式， 并可配合 @placeholder、@loading、@error 子块。

on idle

浏览器空闲时加载

页面完成初始渲染、浏览器进入空闲状态后自动触发，适合非关键内容。

@defer (on idle) { } @loading (minimum 500ms) {
} @placeholder {
等待浏览器空闲…
} 
@defer (on idle) {
  <app-heavy-card />
} @loading (minimum 500ms) {
  <nz-spin nzSimple />
} @placeholder {
  <div>等待浏览器空闲…</div>
}
on interaction

用户交互时加载

点击或按键触发 placeholder 元素时开始加载，适合折叠面板、弹窗等按需内容。

@defer (on interaction(triggerBtn)) { } @loading (minimum 500ms) {
} @placeholder { } 
@defer (on interaction(triggerBtn)) {
  <app-heavy-card />
} @placeholder {
  <!-- triggerBtn 是触发元素，点击/键盘触发 -->
  <button #triggerBtn nz-button>点击触发</button>
}
on hover

鼠标悬停时加载

鼠标移入 placeholder 区域时触发，适合 tooltip、预览卡片等悬停内容。

@defer (on hover(hoverZone)) { } @loading (minimum 500ms) {
} @placeholder {
将鼠标移入此区域触发加载
} 
@defer (on hover(hoverZone)) {
  <app-heavy-card />
} @placeholder {
  <!-- hoverZone 是触发元素，mouseenter 触发 -->
  <div #hoverZone>将鼠标移入此区域</div>
}
on timer

定时延迟加载

页面渲染后等待指定时间再加载，适合广告位、次要信息等低优先级内容。

@defer (on timer(3s)) { } @loading (minimum 500ms) {
} @placeholder (minimum 1s) {
3 秒后自动加载…
} 
@defer (on timer(3s)) {
  <app-heavy-card />
} @placeholder (minimum 1s) {
  <!-- minimum 保证 placeholder 至少显示 1s -->
  <div>3 秒后自动加载…</div>
}
when

条件满足时加载

绑定一个布尔表达式，值变为 true 时触发，适合权限控制、步骤流程等场景。

@defer (when showWhen()) { } @loading (minimum 500ms) {
} @placeholder {
等待条件满足（showWhen = false）…
} 
// TS
readonly showWhen = signal(false);

// Template
@defer (when showWhen()) {
  <app-heavy-card />
} @placeholder {
  <div>等待条件满足…</div>
}
on viewport

进入视口时加载

元素滚动进入可视区域时触发，是长列表、图片懒加载的最佳选择。

@defer (on viewport) { } @loading (minimum 500ms) {
} @placeholder {
滚动到此处触发加载…
} 
@defer (on viewport) {
  <app-heavy-card />
} @placeholder {
  <!-- placeholder 本身即观察目标 -->
  <!-- 基于 IntersectionObserver 实现 -->
  <div>滚动到此处触发加载…</div>
}
prefetch

预取 + 延迟渲染（prefetch on idle; on interaction）

prefetch 控制 JS chunk 的下载时机，触发条件控制渲染时机，两者独立。 此示例在浏览器空闲时预下载代码，但等到点击时才渲染，兼顾性能与体验。

@defer (on interaction(prefetchBtn); prefetch on idle) {
} @loading (minimum 300ms) {
} @placeholder { } 
@defer (
  on interaction(prefetchBtn);  // 渲染时机：点击时
  prefetch on idle              // 下载时机：浏览器空闲时
) {
  <app-heavy-card />
} @placeholder {
  <button #prefetchBtn>点击渲染</button>
}

// prefetch 支持所有触发条件：
// prefetch on idle | viewport | hover | timer
