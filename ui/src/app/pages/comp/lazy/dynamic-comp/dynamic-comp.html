ViewContainerRef.createComponent()

在运行时动态创建组件并插入 DOM，通过 setInput() 注入数据，通过 instance.output.subscribe() 监听事件，通过 ref.destroy() 销毁。

已创建 {{ dynCards().length }} 个

每次点击「创建组件」，都会在下方容器中动态插入一个新的 DynCard 组件实例。点击卡片上的「销毁」按钮可单独移除。 

// 1. 获取宿主容器
const vcr = viewChild<ViewContainerRef>('dynHost',
  { read: ViewContainerRef });

// 2. 动态创建组件
const ref = vcr().createComponent(DynCard);

// 3. setInput() 注入数据（触发 ngOnChanges）
ref.setInput('title', '动态卡片');
ref.setInput('theme', 'blue');

// 4. 订阅 output
ref.instance.destroy.subscribe(() => ref.destroy());

// 5. 销毁
ref.destroy();

ComponentPortal + CdkPortalOutlet

CDK Portal 将组件包装为可传送的内容单元，通过 [cdkPortalOutlet] 渲染到任意位置。 (attached) 事件返回 CdkPortalOutletAttachedRef，可调用 setInput() 并订阅 output。

// 1. 创建 Portal（包装组件）
const portal = new ComponentPortal(PortalCard);
this.selectedPortal.set(portal);

// 2. 模板中声明 outlet
<ng-template
  [cdkPortalOutlet]="selectedPortal()"
  (attached)="onAttached($event)"
/>

// 3. attached 回调中注入数据
onAttached(ref: CdkPortalOutletAttachedRef) {
  if (!(ref instanceof ComponentRef)) return;
  ref.setInput('title', '步骤内容');
  ref.instance.next.subscribe(() => {
    ref.destroy();       // 卸载当前
    this.goNext();       // 切换到下一步
  });
}
Portal 适合
  • 需要将内容渲染到任意 DOM 位置（如 overlay）
  • 内容与宿主模板解耦
  • 配合 Dialog / Drawer 等场景
createComponent 适合
  • 在固定容器内动态增删组件
  • 需要精确控制组件生命周期
  • 列表式动态渲染（如广告位轮播）
